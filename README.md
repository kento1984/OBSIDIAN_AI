# OBSIDIAN_AI

Obsidianを使用した日報・週報管理システムです。AIアシスタントと連携して、効率的に日次・週次の業務報告を作成・管理できます。

## 📋 概要

このプロジェクトは、Obsidianのマークダウンベースのノート機能を活用し、構造化された日報・週報を作成・管理するためのシステムです。5W1H（What, Why, When, Where, Who, How）を意識したテンプレートを使用し、業務内容を体系的に記録します。

## 🗂️ プロジェクト構造

```
OBSIDIAN_AI/
├── 00_sample/          # サンプルファイル
├── 01_Daily/           # 日次ファイル
├── 02_Inbox/           # 受信箱（一時保存）
├── 03_Template/        # テンプレート
├── 04_Docs/            # ドキュメント
├── 05_Personal/        # 個人メモ
├── 06_Apology/         # 謝罪文テンプレート
├── 07_Output/          # 出力ファイル（週報、分析結果など）
├── data/               # データファイル（祝日データなど）
└── .cursor/rules/      # Cursor AI設定
    ├── input/          # 入力テンプレート
    └── output/         # 出力ファイル（日報、週報）
```

## ✨ 主な機能

### 日報管理
- **5W1H形式の構造化テンプレート**: 業務内容を体系的に記録
- **成果・課題の明確化**: 本日の成果と課題を分けて記録
- **振り返り機能**: 学んだこと・気づきを記録
- **翌日予定の管理**: 明日の予定を事前に整理

### 週報作成
- **Marp形式のプレゼンテーション資料**: 週報をMarp形式で自動生成
- **日報の集約**: 一週間分の日報をまとめて週報を作成
- **成果の可視化**: 週単位での成果を整理

### AI連携
- **Cursor AIとの連携**: AIアシスタントによる日報・週報の自動生成
- **テンプレートベースの作成**: 統一されたフォーマットで報告書を作成

## 📝 日報テンプレート

日報は以下の構造で記録します：

- **基本情報**: 日付、担当者、天気
- **本日の業務内容**: 5W1Hを意識した詳細な業務記録
- **本日の成果**: 具体的な成果内容と数値・指標
- **課題・問題点**: 問題の詳細、対応策、今後の対策
- **学んだこと・気づき**: 業務を通じて得た知見
- **所感**: 業務を通じて感じたこと、改善点、提案事項
- **明日の予定**: 翌日の予定と目標
- **その他・連絡事項**: 上司やチームメンバーへの連絡事項

## 🚀 使い方

### 1. 日報の作成

`.cursor/rules/input/日報テンプレ.md`を参考に、日報を作成します。

```bash
# 日報ファイルの例
.cursor/rules/output/2026_01-01_日報.md
```

### 2. 週報の作成

日報を参照して、週報をMarp形式で作成します。

```bash
# 週報ファイルの例
07_Output/weekly_report_2025_12-26_2026_01-01.md
```

### 3. Marpでの表示

週報ファイルをMarpで開いて、プレゼンテーション形式で表示できます。

## 🔧 必要なツール

- **Obsidian**: マークダウンノート管理
- **Cursor**: AIアシスタント連携
- **Marp**: プレゼンテーション資料作成（オプション）
- **Git**: バージョン管理
- **GitHub CLI**: GitHub連携（オプション）

## 📌 重要なルール

このプロジェクトでは以下のルールを遵守します：

1. **ファイルは絶対に削除しない**
2. **インプットのフォルダとアウトプットのフォルダは分ける**
3. **わからないことは確認する**

## 📁 ファイル命名規則

- **日報**: `YYYY_MM-DD_日報.md`
- **週報**: `weekly_report_YYYY_MM-DD_YYYY_MM-DD.md`
- **入力テンプレート**: `.cursor/rules/input/` に配置
- **出力ファイル**: `.cursor/rules/output/` または `07_Output/` に配置
